# 一刻 Yike × 種樹：怎麼 deliver 這個 idea（2026-07-06）

> 背景：種樹專注主題 + 樹林總結已上線 yikeday.com，但**還沒給真人用過**。
> QQ 的顧慮（對的）：要保留「一刻」最棒的樣子，別跟 Forest 混淆、被稀釋。
> 這份文件記錄交付策略，別忘。

## 一句話定位（分家 Forest 的關鍵）

> **Forest 讓你為了種樹而分心地專注。一刻讓你先誠實面對今天最重要的一件事——樹，是你這一年每一個「一刻」長出來的證據。**

- Forest 的故事＝「為了種樹而專注」，樹是誘餌。
- 一刻的故事＝「先想清楚今天最重要的一件事」，樹是**勳章/證據**。
- 樹要表達的是 yike 的**護城河：日→週→月→季→年的縱深**（Forest 沒有）。
  - Forest＝一次專注一棵樹。
  - 一刻＝**一整年的認真，長成一片樹林**。
  - → 所以「樹林總結」才是這功能的靈魂，不是種樹本身。
- **順序鐵律：主打意圖（MIT），樹當證據。** 順序錯了就變成另一個 Forest。

## Deliver 三動作（照順序）

1. **先給 5–10 個真人用，別再加功能。**（最高價值的下一步）
   - 別再為 N=1（自己）打磨。丟連結給 Threads 上做 PKM 的人 + 幾個朋友。
   - 只問一句：「打開後你第一個動作是什麼？卡在哪？」
   - 在看過陌生人怎麼用之前，影片的 hook 都是猜的。
2. **交付載體＝build-in-public 影片，不是功能公告。**
   - 故事：「我用 AI 一個晚上，幫我的手帳長出一片會隨我一年成長的樹林。」
   - 一次做三件事：賣 yike、當 M2 內容、證明「會寫 code 的 Tiago」品牌。
   - 主線用「一年長成樹林」（縱深），不是「種樹很好玩」（Forest）。
3. **Landing / 第一次打開永遠先講 MIT，樹是驚喜不是賣點。**
   - Hero 標題留給「今天就算只做成這一件，也值得了」，別放樹。
   - 種樹是用下去才發現的彩蛋。

## 待 QQ 拍板的決定

樹目前**預設開**。兩條路：
- (A) 維持預設種樹（demo 好看、上線有話題）——但 onboarding 一定先講 MIT。**← 傾向 A**
- (B) 改預設方格、種樹當獎勵選項（保護冷靜誠實的靈魂，想玩的人自己開）。
- 拍板時機：**等 5–10 個真人回饋回來再決定**。

## 暫緩的點子（記著，別現在做）

- **冥想 session**：可做且零著作權風險（禪模式墨圈 + 呼吸節奏視覺 + Web Audio 自生鐘聲/白噪音 + 自寫引導字，全原創）。但**先擱**——現在該做的是給真人用 + 影片，不是加功能。

## 大原則（給未來的自己）

> 這一波是「打磨工具而非出貨內容」的經典陷阱。功能已經夠好了。
> 下一步的槓桿在**真人回饋 + 影片**，不在更多功能。
> 偏離檢測：「這推進 M2（YouTube×2 + 電子報）嗎？」——加功能 No，影片 Yes。
